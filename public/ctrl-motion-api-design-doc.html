Section 1

Overview

Motion Lite is an internal campaign management platform built by Example Co, a digital marketing agency specializing in paid advertising for local service businesses. The platform integrates with the Google Ads API to provide centralized campaign monitoring, performance reporting, and optimization recommendations across all managed client accounts.

Purpose

The Google Ads API integration enables Motion Lite to pull campaign performance data from all client accounts managed under a single Manager (MCC) account. This data powers the platform's reporting dashboards, automated alerting, and optimization recommendations.

Scope of API Usage

  • Read-only access to campaign performance metrics
  • Account enumeration under MCC hierarchy
  • Daily metric aggregation and trend analysis
  • No campaign creation, modification, or budget changes via API

Motion Lite is used exclusively by Example Co staff to manage client advertising accounts. It is not a publicly available product or SaaS offering.

Section 3

API Usage

Endpoints Used

Endpoint / Method Purpose
customers:listAccessibleCustomers Enumerate all client accounts under the MCC hierarchy
googleAds:searchStream Execute GAQL queries to pull campaign, ad group, and keyword performance metrics

Data Fields Retrieved

The following metrics are queried via GAQL (Google Ads Query Language):

  • metrics.cost_micros -- Total spend
  • metrics.impressions -- Ad impressions served
  • metrics.clicks -- Clicks received
  • metrics.ctr -- Click-through rate
  • metrics.conversions -- Total conversions
  • metrics.conversions_value -- Conversion value
  • metrics.cost_per_conversion -- Cost per acquisition
  • Calculated: ROAS (conversions_value / cost)
  • metrics.search_impression_share -- Search impression share

Sync Frequency

Sync Type Frequency Date Range
Automatic background sync Every 4 hours Last 3 days (rolling)
Manual refresh (user-triggered) On demand Last 3-30 days (configurable)

Data Storage

All retrieved data is stored in the Supabase google_ads_daily table with the following schema:

  • Account ID, campaign ID, ad group ID
  • Date (daily granularity)
  • All metrics listed above
  • Campaign type, campaign name, status
  • Sync timestamp

  1. Trigger: Background cron job runs every 4 hours (or user triggers manual refresh)

Section 5

User Interface

The Motion Lite Google Ads dashboard provides a comprehensive view of campaign performance across all managed client accounts. The following describes the key interface components:

Dashboard Summary

The top-level dashboard displays aggregate metrics in a summary bar:

  • Total Spend -- Aggregate cost across all campaigns for the selected date range
  • Clicks -- Total clicks received
  • Impressions -- Total impressions served
  • CTR -- Aggregate click-through rate
  • Conversions -- Total conversion count
  • CPA -- Average cost per acquisition
  • ROAS -- Return on ad spend

Campaign Type Breakdown

Performance is segmented into cards by campaign type:

  • Search -- Text ads on Google Search results
  • Display -- Banner and responsive display ads across Google Display Network
  • Performance Max (PMax) -- AI-driven cross-channel campaigns
  • Video -- YouTube and video partner campaigns
  • Shopping -- Product listing ads (where applicable)

Each card shows spend, clicks, conversions, and CPA for that campaign type.

Visualizations

  • Search Impression Share: Gauge visualization showing share of eligible impressions captured vs. lost (budget and rank)
  • Daily Spend & Conversions: Dual-axis time series chart showing daily spend trend alongside conversion volume

Campaign Performance Table

A sortable data table lists all active campaigns with columns for:

  • Campaign name and type
  • Status (Active / Paused)
  • Spend, impressions, clicks, CTR
  • Conversions, CPA, conversion value, ROAS

Filters

  • Date Range: Preset ranges (Today, 7d, 14d, 30d) and custom date picker
  • Account Selector: Filter by individual client account or view all accounts
Section 6

Security & Privacy

Authentication & Access Control

  • OAuth 2.0 tokens are encrypted at rest in the application database
  • Access to Motion Lite is restricted to authenticated agency users with owner role only
  • No public-facing access to the Google Ads data or API credentials
  • Session management with secure, HTTP-only cookies

Data Handling

  • No client Personally Identifiable Information (PII) is stored beyond account names and campaign identifiers
  • Performance metrics are aggregate-level data (impressions, clicks, costs) with no user-level tracking
  • Data is stored on Example Co' managed infrastructure (Hetzner VPS and Supabase)
  • No data is sold, shared, or transmitted to third parties

Section 7

Rate Limiting & API Compliance

Request Throttling

  • Automatic sync cooldown: Minimum 4-hour interval between background sync cycles
  • Manual refresh cooldown: Rate-limited to prevent excessive API calls
  • Per-account queuing: Accounts are queried sequentially, not in parallel, to avoid burst traffic

Query Optimization

  • Queries are limited to active campaigns only (filters out removed and paused campaigns)
  • Date ranges are kept minimal (3-day rolling window for automatic syncs)
  • Results are cached locally to prevent redundant API calls

Error Handling

Scenario Behavior
Rate limit (429) Exponential backoff starting at 5 seconds, max 3 retries
Auth failure (401) Attempt token refresh; alert user if refresh fails
Server error (5xx) Retry with backoff; skip account and continue with others
Quota exceeded Halt sync cycle; resume at next scheduled interval

Motion Lite's API usage patterns are designed to stay well within Google Ads API rate limits and quotas. The platform prioritizes minimal, efficient queries to reduce API load.
