# Motion Lite / Motion Lite — Design Context

Motion Lite is Operator's internal ops tool (ads agency + content studio + personal assistant). Single-operator power tool, not a SaaS dashboard. It's used daily, for hours, across browser tabs, on multiple monitors, and on mobile when stepping away.

Aesthetic north star: **precision instrument, not dashboard decoration.** Ops-tool calm. Bloomberg terminal sensibility with a designer's restraint. The absence of decoration is the statement.

## Design Context

### Users
- **Primary:** Operator — founder of Example Co, solopreneur. Fast, impatient, values density over whitespace, lives in dark mode. Hours per day.
- **Secondary:** Jimmy (the AI assistant) reads/writes via this interface. Needs legible dense data more than delight.
- **Tertiary:** Occasional client portal usage (read-only, lighter weight).

### Brand Personality
- **Three words:** precise, calm, direct.
- Not "modern," not "elegant," not "playful." Those are empty categories.
- Feels like a CLI made visible. Operator, not showcase.
- Confidence expressed through restraint. Nothing decorative survives.

### Aesthetic Direction
- **Type:** Geist Sans everywhere. **JetBrains Mono** only for all-caps structural labels (`FAVORITES`, `VIEWS`, column headers, timestamps, code, mono values). One family for body.
- **Accent (both themes):** Claude terracotta orange `#D97757` for primary CTAs, focus rings, active rows, selection. White text on the button. Rare by design — overuse kills the signal.
- **Sage green `#7a9e87`:** demoted from brand to `--status-completed` only. Amber `#d9a040` = `--status-active`. Red-coral = overdue. Three status colors; nothing else colored unless semantically meaningful.
- **Neutrals:** warm-tinted darks (`#1c1e1c` bg, `#121311` chrome, `#2a2c29` surface, `#333532` elevated, `#212320` panel). Layers are distinct enough to read as depth. 5% white overlay for hover.
- **Light theme:** warm paper (`#f2efe8` bg, `#e8e4d9` chrome, `#fbf8f1` surface, `#ffffff` reserved for modals only). Shadows tinted warm, not cool black.
- **Depth:** 1px inset top highlight on cards/surfaces. Inputs RECESS into their container (`--bg-field` darker than surface). Modals float above a blurred scrim.
- **Logo:** CTRL green icon (sidebar), CtrlStrategies wordmark (page headers). Project/workspace color pickers are user-chosen — never touch.

### Anti-references (explicit bans)
- **Never:** gradient text, colored side-stripe borders (`border-left: 3px solid var(--...)`), glassmorphism as decoration, rounded-rect metric cards with giant numbers, donut/bar chart dashboards, bounce/elastic easing.
- **Accent text on the button:** use `var(--accent-fg)` (`#ffffff`). Do NOT use a random hex or `white` literal.
- **Never:** use `color-cyan-on-dark` or `purple-to-blue gradient` palette. Feels like AI default.
- **Never:** monospace used decoratively to signal "technical." Mono is reserved for labels + data only.
- **Never:** identical tile grids (4 same-sized cards). Break hierarchy with at least one dominant element.

### Design Principles
1. **Token every color.** Never hardcode a hex in a component — `var(--*)` always, so theme swaps are instant.
2. **One primitive per pattern.** Page headers, empty states, skeletons, data tables, buttons, cards — shared components. If a new page needs its own, the primitive is incomplete.
3. **Density over decoration.** Information first. Visual effects second, and only if they sharpen the data.
4. **Warm neutrals, sparse accent.** The accent only appears where action is invited or state matters. Everywhere else is grayscale-with-warmth.
5. **Depth through layers, not shadows.** The difference between `chrome → bg → panel → surface → elevated` carries hierarchy. Shadows are subtle, warm-tinted, and reserved for truly floating elements.
6. **Typography carries structure.** All-caps JetBrains Mono labels at 9-11px = section chrome. Geist Sans at 13-22px = content. Hierarchy via size and weight only, never color tricks.
7. **Universal before specific.** If three pages need the same pattern, build it in `src/components/ui/` and stop writing inline styles for it.

### Working Conventions
- Tokens live in `src/app/globals.css`. Extend there — don't shadow them in a component.
- Primitives live in `src/components/ui/`. Page-specific components live in `src/components/<page>/`.
- Icons: `src/components/ui/Icons.tsx`. One set, 1.5px stroke, size prop in px. Never inline SVGs in new code.
- Accessibility: preserve `:focus-visible` accent glow on every interactive element. Respect `prefers-reduced-motion`.
- Deploy: `git push origin main` → server auto-deploys at `${SERVER_IP_2}:/root/ctrlmotion` → app.example.com.
